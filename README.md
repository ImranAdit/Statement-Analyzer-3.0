# Adit Pay — Statement Analyser (Render Deploy)

Internal tool for Adit team. Upload a merchant/bank statement and compare fees against Adit Pay pricing.
Restricted to @adit.com Google accounts via SSO.

---

```

├── src/
```

## Deploy to Render (Step by Step)

### Step 1 — Push to GitHub
Push this entire folder to a GitHub repo.

### Step 2 — Create a New Blueprint on Render
1. Go to https://dashboard.render.com
2. Click **New → Blueprint**
3. Connect your GitHub repo
4. Render reads `render.yaml` and creates **two services** automatically:
   - `adit-analyser-api` — Python/FastAPI backend
   - `adit-analyser` — Node/Vite frontend (static site)

### Step 3 — Set Environment Variables
In the Render dashboard, go to each service → **Environment** and set:

**Backend service (`adit-analyser-api`)**:
| Variable | Value |
|---|---|
| `GOOGLE_CLIENT_ID` | From Google Cloud Console |
| `GOOGLE_CLIENT_SECRET` | From Google Cloud Console |
| `SESSION_SECRET` | Auto-generated by Render ✓ |
| `BACKEND_URL` | `https://adit-analyser-api.onrender.com` |
| `FRONTEND_URL` | `https://adit-analyser.onrender.com` |

**Frontend service (`adit-analyser`)**:
| Variable | Value |
|---|---|
| `VITE_API_URL` | `https://adit-analyser-api.onrender.com` |

> ⚠️ The `.onrender.com` URLs are assigned after first deploy. Deploy once, copy the URLs, update the env vars, then redeploy.

### Step 4 — Google OAuth Setup
See `GOOGLE_SSO_SETUP.md` for full instructions.

Add these two Authorized Redirect URIs in Google Cloud Console:
- `https://adit-analyser-api.onrender.com/auth/callback`

## Local Development

```bash
# Backend
cd backend
pip install -r requirements.txt
export GOOGLE_CLIENT_ID=xxx
export GOOGLE_CLIENT_SECRET=xxx
export BACKEND_URL=http://localhost:8000
export FRONTEND_URL=http://localhost:5173
export SESSION_SECRET=any-random-string
uvicorn main:app --reload --port 8000

# Frontend (separate terminal, from root)
npm install
npm run dev   # http://localhost:5173
```

## How Auth Works

1. User visits the frontend → sees login page
2. Clicks "Continue with Google" → redirected to `BACKEND_URL/auth/login`
3. Google OAuth flow → callback hits `BACKEND_URL/auth/callback`
4. Backend checks email ends with `@adit.com` → sets session cookie → redirects to frontend
5. Frontend calls `/api/me` on load to check session
6. All `/api/*` routes require a valid session — 401 otherwise
